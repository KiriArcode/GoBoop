# GoBoop — Чеклист для Figma

Подготовка макетов, компонентов и стилей перед переходом на дизайн в Figma.

---

## 6.1. Экраны

| Экран | Что добавить / уточнить |
|-------|-------------------------|
| **Health (Здоровье)** | Карточка Active Case с этапами, секции «Лекарства» и «Процедуры», расписание приёма, дедлайны, кнопка «Принял» |
| **Travel (Поездки)** | Блок выбора страны, чеклист документов (urgent/pending/done/na), кастомные пункты, прогресс-бар готовности |
| **Quick Add** | Новая кнопка «Лечение», форма добавления лекарства/процедуры |
| **Family** | Доска задач с «Кому», лента активности, фильтр «Мои» |

---

## 6.2. Компоненты

| Компонент | Варианты / состояния |
|-----------|----------------------|
| **TreatmentCard** | Лекарство / Процедура; с расписанием; с дедлайном; с кнопкой «Принял» |
| **TreatmentStep** | Выполнено / не выполнено / с дедлайном |
| **DocumentChecklistItem** | done / pending / urgent / na |
| **ProgressBar** | Общий прогресс лечения; прогресс готовности к поездке |
| **ActivityItem** | Иконка + текст + время + кто сделал |

---

## 6.3. Стили (Luxe Tech)

- **Цвета статусов**: urgent — rose/red; pending — amber; done — emerald; na — neutral.
- **Типографика**: заголовки карточек, подписи к времени, дедлайны.
- **Состояния**: hover, active, disabled для кнопок «Принял» и чекбоксов.
- **Responsive**: mobile-first, touch targets ≥ 44px.

---

## 6.4. Дополнительно для Figma

- **Empty State**: нет активного лечения, нет поездок, пустой чеклист.
- **Ошибки**: «Не удалось сохранить», «Нет интернета».
- **Loading**: скелетоны для карточек лечения и документов.
